XAM135 - Layout in Xamarin.Forms

Duration

20 minutes

Goals

The primary goal of this lab is to use a Grid to arrange the views in a UI.

Required assets

The provided Part 03 Resources folder contains a subfolder named Start that contains starter code. The Completed folder has a solution that you can use to check your work. Please make sure you have these folders before you begin.

Steps

Improve the UI

This exercise is analogous to the previous one: you start with the same version of the TipCalculator app where the views are all placed into one vertical StackLayout. Your goal is to use a Grid to improve the UI.

There is no single "correct" answer here. Use your judgment to produce a UI that looks good to you. Below is a screenshot of the Completed project that you can use as a goal.

The screenshot of the Completed project shown above may be sufficient to allow you to complete the lab. If you would like additional guidance, please use the steps below.

  1. Open the starter solution in the Part 03 Resources/Start folder.
  2. Open MainPage.xaml. All your work will be done in this file; there is no need to modify the code-behind.
  3. Add Padding to the page to avoid overlap of the UI and the iOS status bar. The Top value for the Padding should be at least 20.
  4. Change the layout from a StackLayout to a Grid.
  5. Define 7 rows and 2 columns for the Grid. The image below shows the rows and columns used by the Completed project. All the rows are Auto size except row 3 which uses * so it will be allocated all of the extra space. The columns both use * sizing. Note that * is the default; however, the Completed solution uses it explicitly for clarity. The row and column defintions are provided below, hidden behind a button.

    Show Code 

    <Grid.RowDefinitions>
   <RowDefinition Height="Auto" />
   <RowDefinition Height="Auto" />
   <RowDefinition Height="Auto" />
   <RowDefinition Height="*" />
   <RowDefinition Height="Auto" />
   <RowDefinition Height="Auto" />
   <RowDefinition Height="Auto" />
</Grid.RowDefinitions>

<Grid.ColumnDefinitions>
   <ColumnDefinition Width="*" />
   <ColumnDefinition Width="*" />
</Grid.ColumnDefinitions>
  6. Add settings for Grid.Row and Grid.Column to each of the views to assign them to the appropriate cell in the grid. Note that both of these default to 0; however, it is fairly common to assign this value explicitly for clarity. The code below shows an example of this syntax. 
    <Slider Grid.Row="4" Grid.Column="0" ... />
  7. Align the "Bill" Label and the Entry by setting VerticalOptions to Center on the Label (see below). 
    <Label Text="Bill" VerticalOptions="Center" ... />
  8. Add a setting for Grid.ColumnSpan to the Slider so it spans both columns. 
    <Slider ... Grid.ColumnSpan="2" ... />
  9. Locate the Label with text of "Tip Percentage". Set it so it will occupy the bottom-left spot inside its rectangle (see below). 
    <Label Text="Tip Percentage" VerticalOptions="End" HorizontalOptions="Start" ... />
  10. Locate the Label with a name of tipPercent. Set it so it will occupy the bottom-right spot inside its rectangle (see below). 
    <Label x:Name="tipPercent" VerticalOptions="End" HorizontalOptions="End" ... />
  11. Run the app to test your work.

Summary

In this exercise, you used a Grid to improve the aesthetics of an existing UI. Grid is more powerful than StackLayout; in particular, Grid makes it easy to align views across different rows.
