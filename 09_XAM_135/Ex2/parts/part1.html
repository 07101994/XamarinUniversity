XAM135 - Layout in Xamarin.Forms

Duration

10 minutes

Goals

The goal of this lab is see the effect of the four primary layout options when applied to a view contained in a StackLayout. You will be running a pre-built app; there is no code to write.

Required assets

The provided Part 01 Resources folder contains a subfolder named Start that contains the app you will run. Please make sure you have this folder before you begin.

Steps

Run the provided app

The provided app has 8 buttons and a label. Four buttons change the HorizontalOptions on the label and four buttons change the VerticalOptions. The Label has a gray background so you can determine its size (see below).

At startup, the Label will have the default alignment option of Fill for both horizontal and vertical. It also has a WidthRequest and HeightRequest of 150. Your goal is to run the app and see how the different layout options change the size and position of the Label.

  1. Open the Alignment app in the provided Start folder.
  2. Run the app on a platform of your choice.
  3. At startup, with the default alignment of Fill, are the WidthRequest and HeightRequest settings respected? The Label is in a vertical StackLayout so you should see that the WidthRequest is ignored while the HeightRequest is respected.
  4. Try each of the vertical layout options and observe the behavior. You should not see any change to the Label: the StackLayout is using the HeightRequest and completely ignoring the vertical options.
  5. Try each of the horizontal layout options and observe the behavior. You should see both the alignment and the size update. With Fill, the Label will occupy the full width of the StackLayout so the alignment is irrelevant since it completely fills the horizontal space. With Start, Center, and End, the WidthRequest will be respected and the alignment will also change since the label no longer occupies the full width of the StackLayout.

Summary

There are a few interesting points to take away from this exercise. First, notice that the alignment options (Start, Center, End, and Fill) can change both the size and the alignment of a view. Second, the container can overrule the view's settings; for example, the vertical StackLayout completely ignores the VerticalOptions of the Label. Third, the behavior might be different if we used a different layout such as a Grid.
