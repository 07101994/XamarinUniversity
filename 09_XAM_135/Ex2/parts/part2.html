XAM135 - Layout in Xamarin.Forms

Duration

20 minutes

Goals

The primary goal of this lab is to use nested StackLayouts to arrange the views in a UI.

Required assets

The provided Part 02 Resources folder contains a subfolder named Start that contains starter code. The Completed folder has a solution you can use to check your work. Please make sure you have these folders before you begin.

Steps

Explore the starter code

The starter solution contains a fully functional "Tip Calculator" app. Your goal in this first part is simply to explore the app.

  1. Open the starter solution in the Part 02 Resources/Start folder.
  2. Run the app on a platform of your choice.
  3. Enter a number into the Entry control - the app should then look similar to the image shown below:
  4. Work with the app for a minute or two to get a feel for its behavior.
  5. Stop the app.
  6. Open MainPage.xaml.
  7. Notice that all of the views are placed into one vertical StackLayout.
  8. If you would like to, feel free to open MainPage.xaml.cs and look at the code; however, you will not need to modify the code-behind during the exercise.

Improve the UI

Your goal is to use nested StackLayouts to improve the UI of the Tip Calculator app.

There is no single "correct" answer here. Use your judgment to produce a UI that looks good to you. Below is a screenshot of the Completed project that you can use as a goal.

The screenshot of the Completed project shown above may be sufficient to allow you to complete the lab. If you would like addition guidance, please use the steps below. Run the app after each step so you can see the effect.

  1. Add Padding to the page to avoid overlap of the UI and the iOS status bar. The Top value for the Padding should be at least 20. The code shown below will yield a uniform Padding of 40 units: 
    <ContentPage ... Padding="40">
  2. Use a horizontal StackLayout to group the Label that says "Bill" with the Entry field as shown below: 
    <StackLayout Orientation="Horizontal">
   <Label Text="Bill" ... />
   <Entry ... />
</StackLayout>
  3. Use a horizontal StackLayout to group the Label that says "Tip" with the Label named tipOutput.
  4. Set a Margin on the StackLayout you just added. The top should be 20 units while the left, right, and bottom values should be 0 (see below). 
    <StackLayout Orientation="Horizontal" Margin="0,20,0,0">
   ...
</StackLayout>
  5. Use a horizontal StackLayout to group the Label that says "Total" with the Label named totalOutput.
  6. Use a horizontal StackLayout to group the Label that says "Tip Percentage" with the Label named tipPercent.
  7. Set a VerticalOptions value of EndAndExpand on the StackLayout you just added (see below). 
    <StackLayout Orientation="Horizontal" VerticalOptions="EndAndExpand">
   ...
</StackLayout>
  8. Use a horizontal StackLayout to group the Button with text of "15%" and the Button with text "20%".
  9. Use a horizontal StackLayout to group the Button with text of "Round Down" and the Button with text "Round Up".
  10. Set a HorizontalOptions value of CenterAndExpand on all four of the Buttons (see below). 
    <Button HorizontalOptions="CenterAndExpand" ...>

Summary

In this exercise, you used nested StackLayouts to improve the aesthetics of an existing UI. StackLayout is the simplest layout panel, yet it is powerful enough to produce a reasonable UI.
